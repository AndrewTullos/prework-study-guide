Git

  • git status: checks what branch we are currently on
  • If you are not on the main branch, run "git checkout main" to switch to the main branch.
  • git checkout -b branch-name: creates a new branch and switches to it
  • "cd" changes the directory; similarly ".." walks back the directory one folder.

    • Saving Our Work via Command Line/GitHub!

    1. git add -A
      2. Ensure you are working on the correct branch. Stage the changes.
    2. git commit -m "added CSS to project"
      3. Now that all of our changes have been staged, we can commit to our repository.
    3. git pull origin main
      4. Next, let's make sure our current branch is up to date before opening a pull request.
    4. git push origin feature/add-css
      5. Push up all of our changes
    5. Now navigate to the correct repo in Github. Click the "Pull Requests" tab and then the "New pull request" button.
    6. Make sure the base branch is main and the comparison branch is the feature/add-css branch. Add a title for the pull request along with a short description of the changes, as shown in the following screenshot:
    7. Click the "Create pull request" button.
    8. Now we can merge the comparison branch into the base branch by selecting the "Merge pull request" button and then confirming the merge request.
    9. Great! Now we can close the GitHub issue for CSS. Go to the "Issues" tab and click the GitHub issue for CSS. Click the "Close issue" button. Now you should only see three issues left!
